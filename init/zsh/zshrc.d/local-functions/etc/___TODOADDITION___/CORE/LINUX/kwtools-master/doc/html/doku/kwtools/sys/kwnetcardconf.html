kwnetcardconf

Mit kwnetcardconf kann man alle vorhandenen Netzwerkkarten konfigurieren einschliesslich des Standard Gateway. Bei der Konfiguration des WLAN kann man zusätzlich die Parameter der Verbindung einstellen.

Beim Start des Scripts wird zuerst nach vorhandenen Netzwerkkarten und deren Treiber gesucht. Werden Karten gefunden werden sie folgendermaßen dargestellt:
1 Device_Beschreibung(Netzwerk-Karte)_MAC-Adresse_Treiber
2 Device_Beschreibung(WLAN-Karte)_MAC-Adresse_Treiber
...

Nach einer Auswahl der Karte wird eine Konfigurations Auswahl angeboten, ob man diese über dhcp (automatische Vergabe einer IP-Adresse, etc.) oder manuell vornehmen möchte. Bei WLAN wird gefragt ob man erst die Treiber Parameter einstellen möchte. Bei nein wird mit der Netzwerk Konfiguration begonnen.

Notiz: Bedenken Sie das die Verschlüssellung in der WLAN Konfiguration eingestellt wird.

Notiz: pppoe (*DSL) (über die jeweilige Interface Konfigurationsdatei) wird noch nicht unterstützt. Desweiteren konfiguriert kwnetcardconf nur client seitig. Heißt, es werden keine Server (dhcpd, waproamd, aeswepd,...) konfiguriert.

1 allgemeine Konfiguration

Diese Konfiguration gilt für alle Karten.

IP: Geben Sie hier die IP Adresse Ihrer Netzwerkkarte ein (Bsp. 192.168.0.1)

Boot: Soll das Interface beim Systemstart aktiviert werden (ja/nein)?

Broadcast: Eingabe der Broadcast Adresse (Bsp. 255.255.255.0). Ist optional.

Netmask: Eingabe der Netzmaske (Bsp. 192.168.0.255). Ist optional.

Gateway: Hier gibt man die IP Adresse des Gateway ein. Es wird geprüft, ob dieser Rechner erreichbar ist. Wenn ihr Netz einen Gateway besitzt, dieser aber auf ping Pakete nicht antworten darf, dann müssen Sie diesen von Hand eintragen. Bei Debian + kwtools-net ist das die Datei /etc/network/interfaces.

Nameserver: Eingabe von bis zu 3 Nameservern.

Notiz: Die Nameserver werden aus der Datei /etc/resolv.conf gelesen. Man muss sie also meist nur einmal vergeben.


2 WLAN Konfiguration

Einstellungen für die WLAN Karte:

ESSID: Geben Sie hier einen Namen oder den Namen des Netzwerks ein.

Netzwerk_ID: Geben Sie die Netzwerk ID ein, so das alle angrenzenden Funknetze sich das gleiche Medium teilen können.

Modus: Auswahl des Operations Modus dieser Node (siehe man 8 iwconfig).

Kanal: Geben Sie die Kanal Nummer ein, auf dem gesendet/empfangen (1...16) wird.

Frequenz: Geben Sie die Frequenz Nummer ein + Masseinheit ein. (Bsp. 2.4G = 2,4 GHz bei IEEE 802.11b/g) ein.

Empfindlichkeit: Geben Sie die Empfindlichkeit der Verbindung ein (Bsp. -80, 2; siehe man 8 iwconfig).

Bitrate: Geben Sie die Geschwindigkeit der Datenübermittlung (Bsp. 5.5M = 5.5 MByte) ein.

Schlüssel: Hier kann man die Verschlüssellung auswählen beziehungsweise einen Schlüssel eingeben.

RTS_Handshake: Geben Sie die Grösse für das kleinste Paket (Bsp. 512) ein für welches ein RTS gesendet werden soll. Dies ist ein Overhead der bei einer grossen Anzahl oder versteckten Noden sich ausgleichen kann.

Fragment_Grösse: Geben Sie hier die Grösse ein ab dem ein Paket geslittet werden soll (Bsp. 512).

iwspy_Optionen: Geben Sie hier die Parameter für "iwspy" (siehe man 8 iwspy) ein. "iwspy" zeigt Statistiken über die WLAN Karte.

iwpriv_Optionen: Geben Sie hier die extra (private) Parameter für "iwpriv" (siehe man 8 iwpriv) ein. "iwpriv" stellt treiberspezifische Parameter ein.
