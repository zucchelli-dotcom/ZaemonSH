kwproxy

kwproxy setzt die globalen Umgebungsvariablen http_proxy und ftp_proxy (Debian) und HTTP_PROXY bzw. FTP_PROXY (rpm-basierte Betriebssysteme). unter Debian werden sie in der Datei /etc/default/proxy gespeichert. Bei allen anderen unterstützten Betriebssystemen werden sie in der Datei /etc/sysconfig/proxy gespeichert. Sie sollten drauf achten das das Betriebssystem die Variablen auch einliest, z. B. über die Dateien /etc/{profile,zsh/zprofile}. Diese werden nicht verändert und müssen manuell editiert werden. Der folgende Eintrag sollte dort vorgenommen/vorhanden sein.

if [ -f /etc/default/proxy ] ; then
source /etc/default/proxy
fi

1 Konfiguration

Hier kann man die Proxy Variablen http_proxy und ftp_proxy anlegen beziehungsweise ändern.

Format: Protokoll://host.domain.de:Port, Protokoll://host:Port oder Protokoll://IP:Port.

Beispiel: http://hallo.domain.de:3128 (3128 = Proxy squid)

Um diese Variablen nutzen zu können müssen Sie die Datei, zum einlesen in /etc/profile, /etc/zprofile oder /etc/zsh/zprofile (zsh) eintragen. Die Datei /etc/profile ist für alle shells zuständig. Bei einigen Betriebsystemen (meist rpm-basierte) werden die Daten von einem Init-Script ausgelesen.

NOTIZ: Wenn Sie Debian benutzen achten Sie darauf das die Werte dieselben sind wie in Ihrer /etc/apt/apt.conf.
