kwfuse

kwfuse ist ein Frontend Script fuer FUSE. FUSE (Filesystem in Userspace) ist ein simples Interface fuer Userspace Programme, um virtuelle Dateisyteme in den Linux kernel zu exportieren. FUSE bietet also eine sichere Methode für nicht privilegierten Userzugriff eigene Dateisysteme zu erstellen und diese auch durch eigene Dateisytemimplementationen einzubinden.

kwfuse konfiguriert die Hauptkonfigurationsdatei /etc/fuse.conf und erstellt/löscht Einträge in der Dateisystemtabelle /etc/fstab.

1 Konfiguration

Konfiguration der /etc/fuse.conf Datei.

mount_max: Maximale Anzahl von Dateisystemeinbungen, durch nicht root-Benutzer (Standard ist 1000).

user_allow_other: Dürfen nicht root-Benutzer die Einbinde-Optionen 'allow_other' oder 'allow_root' benutzen (ja/nein)?


2 fusetab_bearbeiten

Hier kann man die Dateisystemtabelle /etc/fstab, für FUSE verwalten (man 5 fstab). Man kann sich die Einträge ansehen, anlegen, ändern und löschen.

Laufwerk: Geben Sie hier das Mountprogramm + Laufwerk ein welches Sie einbinden wollen, getrennt durch ein Rautezeichen (Bsp. ntfsmount#/dev/hda1).

Mountpunkt: Eingabe des Verzeichnis, wo das Dateisystem eingehängt werden soll. Wenn es noch nicht existiert wird es erstellt.

Dateisystem: Welchen Dateisystemtyp besitzt das Laufwerk (Bsp.: fuse).

Mountoptionen: Auswahl der Mountoptionen für das Laufwerk. Dazu gibt es eine eigene Hilfe in /usr/share/doc/fuse-utils/README.gz.

Dump: Soll das Laufwerk gedumpt werden (yes/no)?

Check: Soll das Laufwerk beim Systemstart, mittels fsck (filesystemcheck) überprüft werden (ja/nein)?


3 Laufwerke_ein_ausbinden

Hier kann man ein oder mehrere Konfigurationseinträge, aus der /etc/fstab Datei auswählen und ins Dateisystem einbinden beziehungsweise ausbinden.

NOTIZ: Zur Zeit geht nur Dateisystemtyp fuse.
