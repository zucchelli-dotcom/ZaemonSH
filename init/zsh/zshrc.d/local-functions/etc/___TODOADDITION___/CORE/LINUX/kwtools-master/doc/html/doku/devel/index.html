Hier werden alle Funktionen beschrieben, die von allen Scripts der kwtools der Release Reihe 0.8 benutzt werden.

Notiz: Die Funktionen der kwtools sind immer noch "Work in Process", aber schon soweit das man sie veröffentlichen kann. Es sind einige Funktionen hinzugekommen, umbenannt oder gelöscht worden. Die hauptsächlichste Arbeit bestand an der Verbesserung der Funktionen, wie z.B. unnötigen Code entfernen oder zu optimieren.

Desweiteren wurde auch an der Anpassung von neuen Software/Kernel Features gearbeitet bzw. vorbereitet. Dazu gehört das vergrössern von RAID1 -> RAID5/6, ab "mdadm" Release >=3.1.1 und kernel Release >=2.6.33.

1. script_init = Funktion die von allen Scripts benutzt werden muss und alles nötige initialisiert.

2. fs_conf = Funktionen für Dateien und Dateisysteme.

3. hw_conf = Funktionen die die Hardware betreffen.

4. multi_conf = Funktionen für Brenneinstellungen etc..

5. net_conf = Funktionen die das Netzwerk betreffen.

6. sys_conf = Funktionen die die meisten Scripts benötigen.
Wird von allen Scripts mittels der Funktion "script_init" eingelesen.
