kwlogrotate

Konfiguriert die Rotation und Archivierung von Logdateien einzelner Services. Dabei wird für jeden Service ein Konfigurationsdatei angelegt.

Notiz: Benötigt root Rechte. Konfigurationsdateien von Programmpaketen sollte man nicht ändern, da kwlogrotate nicht alle Optionen unterstützt.

1 Konfiguration

Eintrag hinzufügen: Wenn Sie schon eine Logdatei Konfiguration in dieser Datei (Name) besitzen oder gerade erstellt haben und dazu noch weitere Logdateien mit anderen Regeln, in derselben Datei, einstellen möchten, dann können Sie hiermit neue Einstellmöglichkeiten hinzufügen.

Name: Name der Logrotatedatei. Das sollte normalerweise der Name des Services sein (Bsp. Paket=squid, dann Logrotatename=squid). Dient zur besseren Übersicht.

Logdatei: Auswahl der Logdateien, die rotiert werden sollen.

Anzahl: Anzahl der Rotierungen.

Rotierzeit: Wann soll rotiert werden. täglich, wöchentlich oder monatlich?

Mail: An wen soll im Anschluss eine Mail geschickt werden?

Grösse: Ab welcher Grösse soll rotiert werden?

Kompression: Auswahl des Kompressions-Programms. Es werden nur die installierten Kompressionsprogramme zur Auswahl angeboten.

Rechte: Mit welchen Rechten soll die neue Logdatei angelegt werden. Zur Zeit nur gzip, bzip2 und compress.

Script: Muss der Dämon neu gestartet werden? Wenn ja wie. Beim rsyslogd kann das z. B. "invoke-rc.d rsyslog reload > /dev/null" sein.
