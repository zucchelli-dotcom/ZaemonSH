kwuser

kwuser erstellt, löscht und verändert Benutzer. Beim anlegen eines Benutzers muss man ein Passwort vergeben. Wenn man einige Einstellungen eines Benutzers ändert braucht man dies nicht tun.

Die Option [ -m | --multi ] ist ein Sonderfall mit der Sie viele Benutzer gleichzeitig anlegen können. Es erscheint dann eine etwas andere Menüauswahl mit den zusätzlichen Punkten Prefix, Prefix_Homeverz. und Anzahl. Die Benutzer werden dann mit der Kombination aus Prefix+Anzahl erstellt.

Notiz: Die Passwörter stehen danach in der Datei /root/passwd-prefix.txt. Die Passwörter werden mittels der Gerätedatei /dev/urandom und dem Programm uuencode gebildet. Diese Datei sollten Sie ordentlich sichern!
Die Datei hat das Format "User Passwort".
Auch sollten Sie bei der Erstellung von mehr als 10000 Benutzern lieber eine Datenbank benutzen. Die Geschwindigkeit ist dann besser.

Notiz: Benötigt root Rechte.

1 Konfiguration

Hier können Sie die Werte konfigurieren die beim anlegen von Benutzern angewendet werden sollen.

Gruppe: Gruppe des Benutzers bestimmen.

Gruppen: Gruppen einstellen zu welchen der Benutzer gehören soll. Wird keine eingestellt wird, kommt die Gruppe users zum Zuge (Standard).

Shell: Eine der installierten Shells, als Standard, zum erstellen von Benutzer Accounts auswählen.

Passwortlänge: Eingabe einer Passwortlänge die beim generieren eines Passwortes benutzt werden soll. Wird nichts angegeben wird eine Länge von 8 Zeichen benutzt.

Verschlüsselung: Auswahl der Verschlüsselungs-Methode, die beim erstellen eines Benutzern Accounts, zum erstellen eines Passworts benutzt werden soll.

Zur Auswahl stehen Blowfish, DES, SHA256, SHA512 und MD5. Je nachdem welche unterstützte Distribution benutzt wird. DES ist Standard unter Debian.

Erstlogin: Zu Einstellungen (Passwortänderung) beim 1. Login auffordern (yes/no)? Standard ist "yes". Bei "no" werden die einschlägigen Dateien für "GNOME", KDE", "Bash" und "Zsh" gelöscht. Mehr wird zur Zeit noch nicht unterstützt.

Notiz: Benötigt das Paket "kwtools-sys-kwpasswd", ausser Sie benutzen die Sourcen.

umask: Die "umask" legt fest, welche Rechte eine neu angelegte Datei/Verzeichnis bekommt oder, besser gesagt, nicht bekommt. Falls Sie die "umask"-Regel auch an anderer Stelle, wie ~/.profile angegeben haben gilt diese nur für die Dateien/Verzeichnisse die beim erstellen des Home-Verzeichnisses angelegt werden.

Quota Einstellungen zum anlegen von Benutzern. Quota bedeutet Plattenplatzbeschränkung. Quota wird nur angelegt wenn das Home Verzeichnis des Benutzers auch auf einer eigenen Partition mit Quota Unterstützung liegt (Bsp.: /home/user = /home Quota Laufwerk).

Block_Hardlimit: Grösse des Block Hardlimits. Als Masseinheit stehen kib,KiB,Kib,mib,MiB,Mib,gib,GiB,Gib,.... zur Verfügung.

Block_Softlimit: Grösse des Block Softlimits. Siehe Block_Hardlimit.


2 Menüpunkte zum anlegen von Benutzern

Benutzer: Wählen Sie einen vorhandene Gruppe aus oder geben einen neuen Gruppen Namen ein.

Benutzer_ID: Hier kann man eine Benutzer ID eingeben oder eine vorhandene verändern. Es wird geprüft, ob Sie schon existiert.

Gruppe: Gruppe des Benutzers bestimmen.

Gruppen: Gruppen einstellen zu welchen der Benutzer gehören soll. Wird keiner eingestellt wird, kommt die Gruppe "users" zum Zuge (Standard).

Kommentar: Kommentar eingeben, wie zum Beispiel den vollen Namen (optional).

Shell: Eine der installierten Shell für den Benutzer auswählen.

Home: Home Verzeichnis des Benutzers einstellen. Standard ist /home/Benutzername.

Passwort: Passwort für den Benutzer eingeben oder ändern (ein Muss, wenn es eine Login Shell gibt). Es müssen mindestens 6 Zeichen sein.

Passwort generieren: Hier können Sie ein Passwort für den zu erstellenden Benutzer generieren.

Verfallsdatum: Geben Sie das Datum ein bis zu dem dieses Benutzerfach, ohne Passwort Änderung, bestehen soll (z.B. 60 - 60 Tage).

inaktiv nach Verfallsdatum: Geben Sie hier eine Zahl, in Tagen ein, nachdem das Benutzerfach engültig gesperrt werden soll. Die Zahl richtet sich nach dem Verfallsdatum. 0 ist gleich unbegrenzt.


3 Menüpunkte beim Start von kwuser mit der Option -m

Prefix: Wählen Sie einen vorhandene Gruppe aus oder geben einen neuen Gruppen Namen ein.

Prefix_Homeverz.: Verzeichnis unter dem die Benutzer erstellt werden sollen.

Anzahl: Anzahl der Benutzer die angelegt werden soll.

Gruppe: Gruppe der Benutzer bestimmen.

Gruppen: Gruppen einstellen zu welchen die Benutzer gehören sollen. Wird keine eingestellt wird, kommt die Gruppe "users" zum Zuge (Standard).

Kommentar: Kommentar eingeben, wie zum Beispiel Schüler Klasse3b (optional).

Shell: Eine der installierten Shell für die Benutzer auswählen.

Verfallsdatum: Geben Sie das Datum ein bis zu dem diese Benutzerfächer, ohne Passwort Änderung, bestehen sollen (z.B. 60 - 60 Tage).

inaktiv nach Verfallsdatum: Geben Sie hier eine Zahl, in Tagen ein, nachdem die Benutzerfächer engültig gesperrt werden sollen. Die Zahl richtet sich nach dem Verfallsdatum. 0 ist gleich unbegrenzt.
