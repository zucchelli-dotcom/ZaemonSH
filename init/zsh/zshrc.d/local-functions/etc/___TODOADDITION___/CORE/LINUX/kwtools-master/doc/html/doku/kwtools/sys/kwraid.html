kwraid

kwraid mit den neuen Features vergrössern und ändern funktioniert erst ab Kernel >=2.6.33 und mdadm >=3.1.1!

Lesen Sie bitte vorher das/die Software-RAID-HOWTO's. http://www.linuxhaven.de/dlhp/HOWTO/DE-Software-RAID-HOWTO.html oder
http://unthought.net/Software-RAID.HOWTO

Notiz: Benötigt root Rechte. Als Programm wird "mdadm" benutzt.

1 Konfiguration

Hier können Sie die Datei mdadm.conf einstellen, wie Standardwerte zum erstellen von Arrays oder an wen eine mail, bei Fehlern, geschickt werden soll.

Standard Erstellung: Hier stellt man die Werte ein die bei der Erstellung eines ARRAY standardmässig benutzt werden sollen.

Mailadresse: Eingabe der Mailadresse eines Benutzers an dem das Monitoring gesendet werden soll.

Mail Absender: Eingabe der Mailadresse eines Benutzers von dem die mail über das Monitoring gesendet werden soll.

Rechnername: Sie können hier die Werte Rechnername, "system" und "ignore" eingeben. Dieser Wert wird zu den Metadaten hinzugefügt.
Rechnername - Eingabe des Rechnernamens.
system - In diesm Fall wird der Systemaufruf "gethostname" benutzt.
ignore - In diesem Fall wird beim auto-assembled von Arrays die Überprüfung von HOMEHOST ausgeschaltet.

Programm: Eingabe eines Programms, welches bei einem interessanten Monitoring-Ereignis ausgeführt werden soll.


2 Laufwerke

Hier kann man Software RAID Arrays verwalten. Lesen Sie vorher das Software-RAID-HOWTO (de).

anlegen: Konfiguriert und erstellt neue RAID Arrays, für die RAID Level linear, 0, 1, 4, 5, 6 und 10.

ändern: Hier können Sie RAID Arrays ändern. Z. B. von RAID Level 1 zu RAID Level 6, oder von RAID Level 5 zu Level 1.

Notiz: Benötigt "mdadm" >=3.1.1.

vergrössern: Hier können Sie Laufwerke zu einem Array hinzufügen, dabei wird die Grösse des Dateisystems angepasst.

verkleinern: Hier können Sie Laufwerke von einem Array entfernen, dabei wird die Grösse des Dateisystems angepasst.

Reparatur: Hier wird versucht defekte Arrays zu reparieren.

Partitionswechsel: Hier kann man defekte Festplatten/Partitionen austauschen. Das ganze geschieht im laufenden Betrieb. Wenn es keine kaputten Partitionen/Festplatten zum Array gibt werden alle zum wechseln angezeigt. Sie können das Laufwerk entfernen, das neue partitionieren und wieder hinzufügen.

Notiz: Das ganze funktioniert nur, wenn ihre Hardware hotplugfähig ist. Heisst (E)IDE Festplatten gehen garnicht und bei SCSI erst kontrollieren, ob Sie das unterstützen. Es kann sonst zu schweren Hardware Fehlern kommen.

löschen: Löschen von ein oder mehreren Raid Arrays. Der Superblock der dazugehörigen Partitionen wird dabei auch gelöscht.


3 Monitor

Hier kann man einen RAID Monitor einstellen, der per mail einen User benachrichtigt, wenn in einem RAID Array Partitionen/Festplatten ausgefallen/beschädigt sind.

Es wird das Distributions speziefische Administrationstool aufgerufen. Bei CentOS, und Fedora gibt es keines dazu. Hier muss man das Startscript "mdmonitor" aktivieren. Bei OpenSuSE wird "yast2 sysconfig" aufgerufen und bei Debian "dpkg-reconfigure mdadm".


4 Status

Hier kann man 1 oder mehrere Arrays auswählen und sich sehr ausführlich über den aktuellen Zustand, der Arrays, informieren.
