kwrsync

Mit diesem Script können Sie wahlweise entweder ihre Homverzeichnisse mit anderen Rechnern synchronisieren (Laptop) oder Verzeichnisse ihrer Wahl spiegeln. Sie können auch einen crontab Eintrag vornehmen, damit die Verzeichnisse automatisch gespiegelt werden.

1 Konfiguration

Hier konfiguriert man die Zielrechner, Remote Shell, Bandbreite, ob das Home Verzeichnis per cron synchronisiert werden soll und crontab. Bis auf crontab dient diese Konfiguration hauptsächlich der Synchronisation des Home Verzeichnisses.

Zielrechner: Eingabe der IP-Nummern oder Namen der Zielrechner, getrennt durch ein Leerzeichen. Name geht nur wenn dieser Rechner auch bekannt ist (z.B. DNS). Die Erreichbarkeit wird nach der Eingabe geprüft.

Remote_shell: Auswahl der installierten remote Shells (z.B. ssh).

Bandbreite: Geben Sie hier ein Limit für die Bandbreite, in Kbyte/s, bei der Übertragung ein (256=256Kb/s).

Home_Sync_Cron: Soll das Homeverzeichnis auch per Cron synchronisiert werden (ja/nein)?

crontab: Das Script kwrsync_cron in die crontab Tabelle des Users einfügen oder löschen. kwrsync_cron führt rsync aus und schickt bei Fehlern eine Mail.

Notiz: Als Programm wird rsync benutzt. Für das ssh Protokoll sollten Sie sich eine Datei namens ~/.ssh/authorized_keys anlegen (man 1 ssh).


2 Verzeichnisse

Hier kann man die Datei ~/.kwtools/kwrsynctab konfigurieren. Sie enthält alle Parameter für kwrsync um viele verschiedene Verzeichnisse mit anderen Rechnern zu synchronisieren. Sie können diese Datei ansehen, anlegen, ändern und löschen.

Ansicht: Ansicht der Konfigurationsdatei ~/.kwtools/kwrsynctab.

anlegen/ändern:

Sourceverz.: Auswahl eines lokalen Verzeichnisses.

Remoteverz.: In welches Verzeichnis soll syncronisiert werden. Kann lokal wie remote sein. Bei remote müssen Sie auch den Zielrechner angeben.

Zielrechner: Eingabe der IP-Nummern oder Namen der Zielrechner. Name geht nur wenn dieser Rechner auch bekannt ist (z.B. DNS). Die Erreichbarkeit wird nach der Eingabe geprüft.

rsync Argumente: Eingabe von zusätzlichen Argumenten für das Programm rsync. Diese müssen mit einem Leerzeichen getrennt sein (man 1 rsync).

Notiz: Standard rsync Argmunte sind: "-avu --numeric-ids --relative --delete --delete-excluded"

Ausschluss: Dateien/Verzeichnisse die von der Synchronisation ausgeschlossen werden sollen. Diese müssen relativ zum Pfad des Sourceverz. liegen. Bei einem Sourceverz. von /home/user/foo dann ist der Ausschluss einer Datei z. B. foo/bar.

löschen: Ein oder mehrere Einträge löschen

Notiz: Als Programm wird rsync benutzt. Für das ssh Protokoll sollten Sie sich eine Datei namens ~/.ssh/authorized_keys anlegen (man 1 ssh).
